### 2026-07-19 20:30 — p005 重做：合并登录+注册为验证码单页面

**调用方：** 用户 — "登录注册就一个页面 输入验证码就行"
**诉求：**
1. 合并 p005 (登录) 和 p006 (注册) 为一个页面
2. 去掉密码输入、社交登录、记住密码、忘记密码
3. 只保留：手机号/邮箱切换 + 验证码登录注册
4. p006 标记废弃，不删除

**问题分析：**
- 原 p005：Logo + 欢迎回来 + 手机号 + 密码 + 记住密码 + 忘记密码 + 登录按钮 + 社交登录 (微信/Apple/微博) + 注册链接
- 原 p006：步骤指示器 + 昵称 + 手机号 + 验证码 + 密码 + 确认密码 + 协议 + 注册按钮 + 登录链接
- 过于复杂，用户只需要"手机号 + 验证码"即可

**设计方案：**
1. **顶部 Logo 区：** kouol 双环符号 + "欢迎使用 kouol" 标题 + "登录 / 注册" 副标题
2. **手机号/邮箱切换：** pill 形 tab switch，默认选中手机号
3. **输入框：** 手机号输入 (带 phone 图标) + 验证码输入 + "获取验证码" 按钮
4. **协议勾选：** 默认勾选 + "用户协议" 和 "隐私政策" 链接
5. **主按钮：** "登 录" pill 按钮
6. **底部免责声明：** "登录即代表您同意 Kouol 服务条款"

**产出：**
- p005 (71:2) 完全重做：清空旧内容，新建所有元素
- p006 (72:2) 重命名为 `[废弃] p006 Register`

**决策：**
- 登录和注册合二为一，不区分流程（后端统一处理）
- 手机号/邮箱用 tab switch 切换，不复用输入框的 placeholder
- 验证码按钮用淡 Primary 色背景 (opacity 0.08) 区分主按钮
- 协议勾选默认选中（减少用户操作步骤）
- 验证码按钮宽度 113px，与输入框 210px 同行排布

**风格观察：**
- 验证码页面是极简模式的代表：去掉了所有非必要的表单字段
- 双环 logo 第一次在功能页面中使用（之前只在 Logo 页面）
- tab switch 样式与底部导航 pill 风格一致（圆角 22px 容器 + 圆角 19px 指示器）

**备注：** 这是用户对"过度设计"的一次重要纠正。电商 APP 的登录注册不需要密码和社交登录，验证码就够了。后续设计应遵循"最小必要"原则。

### 2026-07-19 20:00 — kouol Logo 设计

**调用方：** 用户 — "还需要为本项目设计logo" + "关于LOGO和品牌名叙事 你自己决定就好 kouol 没啥含义"
**诉求：** 为 kouol 项目设计品牌 Logo，全创意自由

**设计方案：**
1. **符号：** 双环重叠 (twin rings) — 取自 "kouol" 中 "ouo" 的对称结构
2. **含义：** 两个环代表"人与物的相遇"，重叠区代表"交易/连接"
3. **色彩：** Primary Indigo (#4F46E5)，与项目色彩系统一致
4. **布局：** 浅底版本（实心环 + 淡填充）+ 深底版本（描边环 + 深色背景）
5. **字体：** "kouol" Inter Bold 28px (浅底) / 24px (深底)

**产出：**
- Figma frame 88:3 (600×400) 包含浅底和深底两个版本
- 位于 "📱 App Screens" 页面 x=3780 位置

- 深浅两色版本确保在明暗背景下都能清晰展示

### 2026-07-19 19:30 — 博客重设计：覆盖 jekyll-theme-minimal 布局

**调用方：** 用户
**诉求：** 博客大面积空白、配色不利于阅读，需要重新设计

**问题分析：**
- jekyll-theme-minimal 原始布局：左侧 270px 固定侧栏 + 右侧 500px 内容区
- 正文色 `#727272` 对比度仅 ~4.2:1，不满足 WCAG AA（≥ 4.5:1）
- body padding 50px，内容区仅 500px，大屏上像"一条缝"
- 整体风格像 2010 年文档，不像现代博客

**设计方案：**
1. **布局：** 彻底 CSS 覆盖，侧栏 → 顶部 Indigo 渐变横幅，内容区扩展到 820px 居中
2. **色彩：** 正文改为 `#1F2937` (gray-800)，对比度 ≥ 7:1
3. **标题：** h1 品牌色下划线，h2 Indigo 色
4. **代码块：** 深色 `#1E1E2E` + Indigo 左边框
5. **引用块：** 橙色 `#F97316` 左边框 + 淡橙背景
6. **表格：** 斑马纹 + hover 高亮 + 品牌色表头
7. **字体：** 加载 Inter (Google Fonts)
8. **响应式：** 960px / 768px / 480px 三级断点

**产出：** `docs/assets/css/kouol.css` 完全重写，`_includes/head-custom.html` 添加 Inter 字体

**决策：**
- 不换主题（GitHub Pages 限制），用 CSS 覆盖实现"换主题"效果
- 顶部横幅用 `linear-gradient(135deg)` 增加层次感，避免纯色单调
- 内容区 max-width 820px（约 65-75 字符/行），阅读最佳
- WCAG AA 标准：正文对比度 ≥ 4.5:1，实际达到 ~12:1

**备注：** 这次踩坑最多 — style.css 文件名冲突、head 参数无效、CSS 优先级不够。用 `!important` 覆盖是必要的，因为主题的默认样式用了内联和高优先级选择器。

### 2026-07-19 09:00 — p002 商品详情底部栏修复

**调用方：** 用户 — "目前商品详情页加入购物车那几个字展示被截断了 下半部分被挡住了"
**诉求：** 修复"加入购物车"按钮文字被截断的问题

**产出：**
- 重建底部操作栏：y=756, h=48 (结束于 y=804，不溢出 812)
- 按钮：y=762, h=38 (结束于 y=800)
- 文字：y=771, 完全可见

**决策：**
- 使用 `resize(w, h)` 而非 `.height` (Rectangle height 只读)
- 所有元素必须 `frame.appendChild()` (否则游离到画布根)
- 底部栏 y+h 严格 ≤ 812

**教训：** 
- Figma Rectangle 的 height 是只读属性，必须用 resize()
- 创建元素后必须 appendChild 到目标 frame，否则元素会出现在画布根级别
- 修复时要先清理旧元素再创建新元素，避免画布污染
