### 2026-07-19 19:30 — 博客重设计：覆盖 jekyll-theme-minimal 布局

**调用方：** 用户
**诉求：** 博客大面积空白、配色不利于阅读，需要重新设计

**问题分析：**
- jekyll-theme-minimal 原始布局：左侧 270px 固定侧栏 + 右侧 500px 内容区
- 正文色 `#727272` 对比度仅 ~4.2:1，不满足 WCAG AA（≥ 4.5:1）
- body padding 50px，内容区仅 500px，大屏上像"一条缝"
- 整体风格像 2010 年文档，不像现代博客

**设计方案：**
1. **布局：** 彻底 CSS 覆盖，侧栏 → 顶部 Indigo 渐变横幅，内容区扩展到 820px 居中
2. **色彩：** 正文改为 `#1F2937` (gray-800)，对比度 ≥ 7:1
3. **标题：** h1 品牌色下划线，h2 Indigo 色
4. **代码块：** 深色 `#1E1E2E` + Indigo 左边框
5. **引用块：** 橙色 `#F97316` 左边框 + 淡橙背景
6. **表格：** 斑马纹 + hover 高亮 + 品牌色表头
7. **字体：** 加载 Inter (Google Fonts)
8. **响应式：** 960px / 768px / 480px 三级断点

**产出：** `docs/assets/css/kouol.css` 完全重写，`_includes/head-custom.html` 添加 Inter 字体

**决策：**
- 不换主题（GitHub Pages 限制），用 CSS 覆盖实现"换主题"效果
- 顶部横幅用 `linear-gradient(135deg)` 增加层次感，避免纯色单调
- 内容区 max-width 820px（约 65-75 字符/行），阅读最佳
- WCAG AA 标准：正文对比度 ≥ 4.5:1，实际达到 ~12:1

**备注：** 这次踩坑最多 — style.css 文件名冲突、head 参数无效、CSS 优先级不够。用 `!important` 覆盖是必要的，因为主题的默认样式用了内联和高优先级选择器。

### 2026-07-19 09:00 — p002 商品详情底部栏修复

**调用方：** 用户 — "目前商品详情页加入购物车那几个字展示被截断了 下半部分被挡住了"
**诉求：** 修复"加入购物车"按钮文字被截断的问题

**产出：**
- 重建底部操作栏：y=756, h=48 (结束于 y=804，不溢出 812)
- 按钮：y=762, h=38 (结束于 y=800)
- 文字：y=771, 完全可见

**决策：**
- 使用 `resize(w, h)` 而非 `.height` (Rectangle height 只读)
- 所有元素必须 `frame.appendChild()` (否则游离到画布根)
- 底部栏 y+h 严格 ≤ 812

**教训：** 
- Figma Rectangle 的 height 是只读属性，必须用 resize()
- 创建元素后必须 appendChild 到目标 frame，否则元素会出现在画布根级别
- 修复时要先清理旧元素再创建新元素，避免画布污染
