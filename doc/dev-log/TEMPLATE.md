# 开发日志模板（航海日志式 · 三级聚合）

> **核心原则：** 边做边记，逐级汇总。
>
> - 每个操作 → 实时追加到**天日志**
> - 每周末 → 汇总本周天日志为**周日志**
> - 每月底 → 汇总本月周日志为**月日志**
>
> 提取关系：操作记录 ⊂ 天日志 ⊂ 周日志 ⊂ 月日志

---

## 一、天日志模板

**文件位置：** `doc/dev-log/daily/YYYY-MM-DD.md`
**写入时机：** 每个操作发生时立即追加；当天结束时写"当日小结"
**生成方式：** AI 实时写入

---

# YYYY-MM-DD · [一句话主题，当天结束时填]

**当日目标：** [今天打算做什么]
**AI 模型：** [Claude / 其他]

### HH:MM - [简短标题]

- **操作：** [做了什么]
- **目的：** [为什么做]
- **结果：** [成功 / 失败 / 部分成功]
- **详情：** [可选 —— 核心代码片段、关键 prompt]
- **遇到的问题：** [可选 —— 翻车时才写]
  - 现象 / 排查 / 根因 / 解决 / 给未来 AI 的提示

## 当日小结

- **完成：** ...
- **未完成：** ...
- **今日一个教训：** ...

---

## 二、周日志模板

**文件位置：** `doc/dev-log/weekly/YYYY-WNN.md`（ISO 周数）
**写入时机：** 每周日 / 下周一 生成
**生成方式：** AI 从本周的天日志中**提炼**，不是简单复制

---

# YYYY 第 NN 周（MM-DD ~ MM-DD）· [本周主题]

**本周目标：** [周初定下的目标]
**实际完成：** [一句话概括]

## 本周关键成果

- [成果 1]
- [成果 2]

## 本周重要踩坑（Top 3）

> 从本周天日志中挑出最有价值的 3 个坑，提炼要点。每个用 3-5 行讲清楚，详细排查过程指向天日志。

### 坑 1：[标题]

- **出现日期：** YYYY-MM-DD（详见 [天日志](../daily/YYYY-MM-DD.md)）
- **现象 → 根因 → 解决：** 一段话讲完
- **教训：** 一句话

## 本周代码变更统计

```
git log --oneline --since="YYYY-MM-DD" --until="YYYY-MM-DD"
git diff --stat <上周末 commit>..<本周末 commit>
```

- 本周 AI 在哪里翻车多？
- 有什么 prompt / 工作流值得沉淀？

## 三、月日志模板

**文件位置：** `doc/dev-log/monthly/YYYY-MM.md`
**写入时机：** 每月最后一天 / 下月初生成
**生成方式：** AI 从本月周日志中**提炼**

---

> 💡 **写好三级日志的要点：**
> - **天日志**是原始数据，要详细、实时
> - **周日志**是提炼，要抓重点、不要复制粘贴
> - **月日志**是反思，要看趋势、看成长
> - 每一级的价值不同：天日志给"当时的自己"看，周日志给"这周的队友"看，月日志给"未来的自己"看
