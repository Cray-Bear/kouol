# 开发日志模板

> 复制本文件为 `YYYY-MM-DD.md` 作为当天的日志。一天多篇用 `YYYY-MM-DD-02.md`。
> 每篇日志控制在 300–800 行，重点写"踩坑"和"AI 协作经验"，不要堆代码。

---

# YYYY-MM-DD · [一句话主题]

**会话目标：** [这次要做什么]
**耗时：** [大约多久]
**AI 模型：** [Claude / 其他，版本]

---

## 1. 做了什么

按时间顺序列出完成的事项。每条用 1–2 行，不要展开细节。

- [ ] 事项 A
- [ ] 事项 B

## 2. 怎么做的

简述实现思路、选用的方案、关键的 prompt 方向。可以贴核心代码片段（≤ 20 行），其余指向具体 commit。

## 3. 遇到的问题（踩坑）

**最重要的一节。** 每个坑按这个结构写：

### 坑 1：[简短描述]

- **现象：** 报了什么错 / 表现异常
- **排查过程：** AI 是怎么分析的，试了哪些方案
- **根因：** 真正的原因是什么
- **解决方案：** 最终怎么修好的
- **给未来 AI 的提示：** 下次遇到同类问题该怎么问 / 避开

## 4. AI 协作经验

- 这次对话里哪些 prompt 有效，哪些无效
- AI 在哪里犯错了 / 我哪里误导了 AI
- 哪些任务适合交给 AI，哪些需要人工介入

## 5. 文件变更

```
git diff --stat <上一个commit>..HEAD
```

或直接列出关键文件：

- `path/to/file.ets` — 做了什么

## 6. 下次计划

- [ ] 明天/下次要做什么
- [ ] 待解决的问题

---

> 💡 **写好日志的要点：**
> - 踩坑部分要详细到"未来的 AI 读到这篇日志就能避开同样的坑"
> - 代码片段只保留关键片段，其余用 commit hash 引用
> - 诚实记录失败和走弯路的过程，这比成功案例更有价值
