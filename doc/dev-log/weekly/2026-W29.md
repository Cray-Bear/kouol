## 本周重要踩坑（Top 3）

### 坑 1：Bash 工具 ≠ PowerShell

- **出现日期：** 2026-07-18（详见 [天日志](../daily/2026-07-18.md)）
- **现象 → 根因 → 解决：** `Get-ChildItem` 在 Bash 工具中报错 → Claude Code 的 Bash 工具内部是 bash 不是 PowerShell → 改用 `find` 命令
- **教训：** ⚠️ Windows 上有两个 shell：Bash 工具用 bash 语法，PowerShell 工具用 PS 语法。CLAUDE.md 说的"PowerShell 语法"只适用于 PowerShell 工具。

### 坑 2：LF/CRLF 警告

- **出现日期：** 2026-07-18
- **现象 → 根因 → 解决：** `git add` 时大量 LF→CRLF 警告 → 未配置换行符转换 → `git config core.autocrlf true`
- **教训：** Windows 上 git 初始化第一件事就是 `core.autocrlf=true`，避免污染 git history。

### 坑 3：日志机制需要迭代

- **出现日期：** 2026-07-18
- **现象 → 根因 → 解决：** 第一版日志机制是"会话总结式"，用户不满意 → 重新理解为"航海日志式"实时记录 → 再次升级为三级聚合
- **教训：** 当用户说"日志/记录"时，**默认理解为实时流水记录**，不是事后总结。机制设计本身也需要迭代，第一版错了就重写。
