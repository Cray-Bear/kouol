3. 验证部署成功

## 验收标准

- [x] GitHub Pages 已启用
- [x] 构建状态显示成功
- [x] 可以访问 https://cray-bear.github.io/kouol/

## 执行记录

使用 gh CLI 通过 API 完成配置：
```bash
# 启用 Pages
gh api repos/Cray-Bear/kouol/pages -X POST \
  -f "build_type=legacy" \
  -f "source[branch]=main" \
  -f "source[path]=/docs"

# 触发构建
gh api repos/Cray-Bear/kouol/pages/builds -X POST

# 检查状态
gh api repos/Cray-Bear/kouol/pages/builds/latest
```

构建成功，HTTP 200。

## 踩坑

- 首次配置时 build_type 设为 "workflow"，但没有对应的 workflow 文件
- 改为 "legacy" 模式后正常构建
- **教训：** GitHub Pages 有两种模式，legacy 模式更适合静态站点
