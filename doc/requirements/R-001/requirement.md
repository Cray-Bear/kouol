# R-001 需求文档

## 背景

kouol 是一个纯 AI 驱动的教学项目，需要一套完整的协作机制来：

1. 记录 AI 开发的全过程（做了什么、怎么做的、踩了哪些坑）
2. 管理想法 → 需求 → 任务 → 执行的完整流程
3. 支持中断后恢复（打开日志就知道做到哪了）
4. 为其他想用 AI 做开发的人提供参考

## 需求目标

建立一套以需求为中心的协作文档体系，包含：

- **想法管理** — 随手记录灵感，可合并到需求
- **需求管理** — 每个需求是一个目录，包含所有相关文档
- **任务管理** — 需求拆解为可执行的任务
- **开发日志** — 跟着需求走，实时记录进度
- **编号系统** — 不同类型文档有各自的全局编号

## 核心原则

1. **需求是宇宙中心** — 所有文档、任务、日志都围绕需求组织
2. **一切起点是想法** — 想法 → 需求 → 任务 → 执行
3. **边做边记** — 每个操作实时记录
4. **中断可恢复** — 打开 log.md 就知道做到哪了

## 验收标准

- [x] 文档结构清晰（ideas/, requirements/, R-xxx/）
- [x] 编号规则明确（I-xxx, R-xxx, T-xxx）
- [x] 状态流转完整（想法、需求、任务各自的状态）
- [x] 开发日志跟着需求走（log.md 在 R-xxx/ 内）
- [x] 协作机制写入 CLAUDE.md
- [x] 有示范需求（R-001 本身）

## 非目标

- 不追求完美的工具链（markdown 足够）
- 不强制使用外部工具（GitHub Issues 等）
- 不要求实时同步（文件 + git 即可）
