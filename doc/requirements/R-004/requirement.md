# R-004 · P0 主页开发 — 需求文档

## What（做什么）

开发 APP 主页（p001 Home Screen），还原 Figma 设计稿，实现商品浏览功能。

### 功能需求

1. **商品 Grid 展示**
   - 2 列网格布局，展示商品卡片
   - 每个卡片：商品图片 + 名称 + 价格
   - 点击商品卡片 → 跳转商品详情页（p002，本期只做跳转，p002 开发在 P1）

2. **搜索栏**
   - 顶部搜索栏，点击 → 跳转搜索页（暂用空页面占位）

3. **Banner 轮播**
   - 首页顶部 Banner 区域，展示 2-3 张推广图
   - 自动轮播 + 手动滑动

4. **底部导航**
   - 4 个 Tab：首页 / 分类 / 购物车 / 我的
   - 首页 Tab 高亮，其他 Tab 点击暂不跳转（P1 实现）

5. **路由系统**
   - 搭建基础路由框架（为后续页面做准备）
   - 底部导航组件（可复用）

### 数据

- 使用 mock 数据（商品列表、Banner 图片）
- 暂不接入真实 API

## Why（为什么做）

- **治理演练**：用真实需求走一遍完整流程（I→R→T→开发→测试→审查→发布→审计）
- **技术基础**：建立路由、导航、列表组件等基础架构
- **教学价值**：覆盖 ArkUI 核心模式（路由、布局、状态管理、组件复用）

## 验收标准

| # | 标准 | 验证方式 |
|---|------|---------|
| AC-1 | 主页在模拟器上正常运行 | 实机/模拟器验证 |
| AC-2 | 视觉还原 Figma p001 设计稿 | 截图对比 |
| AC-3 | 商品 Grid 展示正确（图片+名称+价格） | 功能测试 |
| AC-4 | Banner 可轮播（自动+手动） | 功能测试 |
| AC-5 | 底部导航组件正确显示，首页 Tab 高亮 | 视觉+功能测试 |
| AC-6 | 点击商品卡片有跳转响应（即使目标页为空） | 功能测试 |
| AC-7 | 治理流程完整记录（discussion/log/test/review） | 审计检查 |
