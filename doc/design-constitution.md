# 设计宪法 (Design Constitution)

> **本文件是 kouol 项目的最高设计准则。** 所有设计决策、页面创建、组件开发都必须遵守本宪法。
> 当其他设计文档与本宪法冲突时，以本宪法为准。

---

## 第一条 · 核心原则

### 1.1 一致性高于创意
所有页面必须使用同一套色彩、字体、间距、组件。不允许"这个页面特殊处理"。

### 1.2 像素级精确
所有元素必须有明确的 x/y/width/height。不允许"大概"、"差不多"。

### 1.3 中文优先
所有界面文字使用中文。价格使用 ¥ 符号。不允许出现英文占位符（Lorem ipsum 等）。

### 1.4 真实数据
使用真实商品名、真实价格、真实地址。不允许 "商品1"、"测试数据"。

### 1.5 移动优先
设计以 375×812 (iPhone 14) 为基准。所有间距、字号、触控区域以此为参考。

## 第三条 · 色彩系统

### 3.1 语义色板 (Semantic Tokens)

```
/* ===== 品牌色 ===== */
Primary:     #4F46E5  indigo-600   → 按钮、价格、选中态、导航高亮、链接
Accent:      #F97316  orange-500   → 促销标签、CTA 按钮、VIP 徽章、限时优惠

/* ===== 中性色 ===== */
BG:          #F9FAFB  gray-50      → 页面背景
Surface:     #FFFFFF  white        → 卡片、输入框背景、底部栏
Text-1:      #111827  gray-900     → 标题、重要文字、价格
Text-2:      #6B7280  gray-500     → 描述、辅助文字、副标题
Text-3:      #9CA3AF  gray-400     → 占位符、禁用态、分割线上的弱文字
Border:      #E5E7EB  gray-200     → 分割线、卡片边框、输入框边框

/* ===== 功能色 ===== */
Success:     #10B981  emerald-500  → 成功状态、已完成、选中勾选
Danger:      #EF4444  red-500      → 删除、错误、警告、移除
Warning:     #F59E0B  amber-500    → 待处理、注意（预留）
Info:        #3B82F6  blue-500     → 提示、链接辅助（预留）
```

### 3.2 色彩使用规则
- **Primary 色**：每个页面最多 3 处大面积使用（按钮 + 导航高亮 + 一个重点元素）
- **Accent 色**：仅用于需要强烈视觉吸引的元素（促销、CTA）
- **Danger 色**：仅用于破坏性操作（删除、移除），不可用于装饰
- **对比度**：文字与背景对比度 ≥ 4.5:1（AA 标准）

## 第四条 · 字体系统

### 4.1 字体家族
- **主字体**: Inter (拉丁/数字)
- **中文字体**: 系统默认 (PingFang SC / Noto Sans SC)
- **禁止**: 使用其他字体族

### 4.2 字号层级

| 层级 | 字重 | 字号 | 用途 |
|------|------|------|------|
| Display | Bold | 27-31px | 大价格、核心数据 |
| Title | Bold | 22-24px | 页面标题 |
| Subtitle | Semi Bold | 17-19px | 栏目名、按钮文字、Tab |
| Body | Regular | 15-17px | 正文、描述、列表项 |
| Caption | Regular | 13-14px | 标签、辅助说明、时间戳 |
| Micro | Regular | 11-12px | 徽章数字、极小标注（少用） |

### 4.3 排版规则
- **行高**: 1.5 倍（body 级别）
- **字间距**: 默认，不手动调整
- **对齐**: 标题居中或左对齐；价格左对齐；列表项左对齐
- **截断**: 长文本用省略号 (...) 截断，不超过 2 行

## 第六条 · 组件规范

### 6.1 状态栏 (Status Bar)
```
位置: y=0, h=44, w=375
背景: 透明 (跟随页面背景)
左侧: 时间 "9:41" — Bold 15px, x=24, y=12
右侧: 信号/WiFi/电池图标 — x=329, y=17
```

### 6.2 导航栏 (Nav Bar)
```
位置: y=44-100, h=56 (内容居中 at y≈62)
背景: 白色或与页面背景一致
返回箭头: 22×22 SVG, x=20, y=62
标题: Bold 19px, 水平居中
右侧操作: 20×20 SVG, x=333
```

### 6.3 底部导航 (Bottom Nav) — 仅主页面
```
位置: y=728, h=64, w=335 (有 20px 左右边距)
背景: Surface 白色, 圆角 16px (可选)
4 个 Tab: 图标 22×22 at y=740, 标签 13px at y=770
图标 x 坐标: 42, 126, 210, 294 (间距 84px)
选中态: Primary 色 + Semi Bold
未选中: Text-2 色 + Regular
```

### 6.4 按钮 (Button)

| 类型 | 高度 | 圆角 | 背景 | 文字 | 场景 |
|------|------|------|------|------|------|
| 主按钮 | 48-52px | 19px (pill) | Primary 色 | 白色 Semi Bold 15-16px | 主要操作 |
| 次按钮 | 48-52px | 19px (pill) | 透明 + 1px Primary 边框 | Primary 色 Semi Bold | 次要操作 |
| 小按钮 | 32-38px | 16-19px | 视上下文 | 13-14px | 卡片内操作 |
| 文字按钮 | auto | 无 | 透明 | Primary 色 Regular | 链接式操作 |
| 危险按钮 | 48-52px | 19px (pill) | Danger 色 | 白色 Semi Bold | 删除/移除 |

### 6.5 卡片 (Card)
```
背景: Surface 白色
圆角: 12-16px
边框: 可选 1px Border 色
内边距: 12-16px
阴影: 不强制，如使用则 0 2px 8px rgba(0,0,0,0.06)
```

### 6.6 输入框 (Input)
```
高度: 48px
圆角: 12px
背景: Surface 白色
边框: 1px Border 色
内边距: 16px (左侧，如有 icon 则 44px)
占位符: Text-3 色 Regular 15px
聚焦态: 边框变为 Primary 色
```

### 6.7 图标 (Icons)
```
来源: Lucide SVG (https://lucide.dev/icons/)
尺寸: 20-22px (标准), 14-16px (小), 24-28px (大)
描边: stroke-width=2, stroke-linecap=round, stroke-linejoin=round
颜色: 跟随上下文 (Primary / Text-1 / Text-2 / Danger)
禁止: emoji 作为图标、自定义几何图形作为图标
```

### 6.8 图片 (Images)
```
格式: JPEG / PNG (真实照片)
来源: Unsplash 或同类高质量图库
商品图: 1:1 比例优先
Banner: 335×170 (全宽), 圆角 12-16px
头像: 圆形裁切
禁止: 灰色占位矩形作为最终展示
```

### 8.4 代码模式

```javascript
// 语义色板
const C = {
  primary: { r: 0.310, g: 0.275, b: 0.898 },
  accent:  { r: 0.976, g: 0.451, b: 0.086 },
  bg:      { r: 0.976, g: 0.980, b: 0.984 },
  surface: { r: 1, g: 1, b: 1 },
  text1:   { r: 0.067, g: 0.094, b: 0.153 },
  text2:   { r: 0.420, g: 0.450, b: 0.500 },
  text3:   { r: 0.612, g: 0.639, b: 0.686 },
  border:  { r: 0.898, g: 0.906, b: 0.922 },
  success: { r: 0.063, g: 0.725, b: 0.506 },
  danger:  { r: 0.937, g: 0.267, b: 0.267 },
};

// SVG 图标工具函数
function colorToHex(col) {
  return '#' + [col.r, col.g, col.b]
    .map(v => Math.round(v * 255).toString(16).padStart(2, '0')).join('');
}
function makeSvg(path, color, size) {
  return `<svg xmlns="http://www.w3.org/2000/svg" width="${size}" height="${size}" 
    viewBox="0 0 24 24" fill="none" stroke="${colorToHex(color)}" 
    stroke-width="2" stroke-linecap="round" stroke-linejoin="round">${path}</svg>`;
}

// 创建元素必须 appendChild 到 frame
const rect = figma.createRectangle();
rect.x = 20; rect.y = 100; rect.resize(335, 48);
rect.fills = [{ type: 'SOLID', color: C.surface }];
frame.appendChild(rect);  // ← 必须！否则元素游离到画布根级别
```

## 第九条 · 交互设计原则

> 参考 UI/UX Pro Max 规则库，以下为本项目必须遵守的核心交互规则。

### 9.1 触控规则 (CRITICAL)
- 最小触控区域: 44×44px
- 触控目标间距: ≥ 8px
- 点击反馈: 100ms 内给出视觉反馈
- 不依赖 hover 状态

### 9.2 导航规则 (HIGH)
- 底部导航 ≤ 5 项
- 返回行为必须可预测
- 每个页面有且只有一个主要 CTA
- 层级深度 ≤ 3 层

### 9.3 反馈规则 (MEDIUM)
- 异步操作显示 loading 状态
- 表单验证在 blur 时触发（非 keypress）
- 错误信息靠近出错字段
- 成功操作给出明确反馈

### 9.4 可访问性基线 (CRITICAL)
- 文字对比度 ≥ 4.5:1
- 图标按钮必须有文字标签或 aria-label
- 颜色不作为唯一信息传递手段
- 支持系统字号缩放

## 第十条 · 反模式 (Anti-Patterns)

**以下做法严格禁止：**

| 编号 | 反模式 | 正确做法 |
|------|--------|----------|
| A-01 | 页面间使用不同主色 | 统一使用 Primary #4F46E5 |
| A-02 | emoji 作为功能图标 | 使用 Lucide SVG |
| A-03 | 灰色矩形占位最终稿 | 使用真实图片 (Unsplash) |
| A-04 | 英文 Lorem ipsum | 使用中文真实文案 |
| A-05 | 元素不 appendChild 到 frame | 必须显式 appendChild |
| A-06 | 底部操作栏溢出 812px | 严格计算 y+h ≤ 812 |
| A-07 | 用 .height 赋值 Rectangle | 使用 .resize(w, h) |
| A-08 | 画布根级别创建游离元素 | 必须在 frame 内 |
| A-09 | 硬编码颜色值到组件 | 使用语义 token (C.primary) |
| A-10 | 一个页面多个主 CTA | 每页只有一个 Primary CTA |
| A-11 | 小于 44px 的可点击区域 | 扩展触控区域到 44px+ |
| A-12 | 仅用颜色传达信息 | 加图标/文字辅助 |
| A-13 | 首页放购物车按钮 | 首页只有浏览/跳转功能 |
| A-14 | 文字截断无省略号 | 单行截断加 "..." |
| A-15 | 无状态栏的页面 | 每个页面必须有状态栏 |

## 第十四条 · 设计日志

> **设计师的航海日志。** 每一次设计调用都必须记录，不可豁免。

### 14.1 日志文件
- **位置**: `doc/design-log.md`
- **格式**: newest first（最新条目在最上面）
- **不可删除**，只可追加

### 14.2 记录义务
设计师 Agent 每次被调用，**必须**在操作完成后写入日志，记录：
1. **调用方** — 谁触发的（用户 / R-xxx 需求 / T-xxx 任务 / 设计师自检）
2. **诉求** — 要做什么、为什么
3. **产出** — 具体产出了什么（页面/修改/决策/文档），涉及哪些 Node ID
4. **决策** — 做了哪些设计决策、为什么这样选
5. **风格观察** — 是否发现可复用的模式
6. **备注** — 教训、踩坑、后续建议

### 14.3 风格备忘
日志顶部维护"设计风格备忘"区域，汇总跨页面的风格模式：
- 色彩使用模式（"Accent 色用于 xxx"）
- 页面结构模式（"列表卡片统一使用 xxx"）
- 图标选择记录（每个功能用了什么 Lucide 图标）
- 图片使用记录（上传了哪些图片、hash）

### 14.4 日志的价值
- **一致性检查**：新页面设计前翻看风格备忘，确保与历史一致
- **决策追溯**：回忆"为什么这个页面这样设计"
- **风格演进**：追踪设计风格随时间的变化
- **团队协作**：其他 AI 或人类可以通过日志了解设计全貌
