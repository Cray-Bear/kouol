# I-007: 补齐缺失状态 — 空状态、加载态、错误态

## 来源

设计师定时巡检（2026-07-19 23:30）扩展巡检发现。

## 背景

现有 9 个页面都只有"正常态"设计，缺少异常场景的视觉方案。

## 缺失清单

### P0 — 空状态（Empty State）

| 页面 | 场景 | 设计要点 |
|------|------|---------|
| p001 首页 | 搜索无结果/无商品 | 插图 + "暂无相关内容" + 返回首页按钮 |
| p003 购物车 | 购物车为空 | 插图 + "购物车是空的" + 去逛逛按钮 |
| p008 收货地址 | 无收货地址 | 插图 + "暂无收货地址" + 添加地址按钮 |

### P1 — 加载态（Loading State）

| 页面 | 场景 | 设计要点 |
|------|------|---------|
| p001 首页 | 数据加载中 | 骨架屏（Banner/分类/商品卡占位） |
| p002 商品详情 | 商品加载中 | 骨架屏（图片区+文字区占位） |
| p009 支付 | 支付处理中 | Loading 动画 + "支付处理中，请稍候" |

### P1 — 错误态（Error State）

| 页面 | 场景 | 设计要点 |
|------|------|---------|
| p002 商品详情 | 商品下架/网络错误 | 插图 + "商品已下架" + 返回首页 |
| p009 支付 | 支付失败/超时 | 图标 + 失败原因 + 重试按钮 |

### P2 — 交互态

| 页面 | 缺失 | 说明 |
|------|------|------|
| p005 登录 | 验证码倒计时态 | "获取验证码"→"60s 后重新获取" |
| p005 登录 | 表单验证错误态 | 手机号格式错误提示 |

## 验收标准

- [ ] 每个空状态有插图 + 文案 + 行动按钮
- [ ] 加载态使用骨架屏而非转圈
- [ ] 错误态有明确的恢复路径
- [ ] 与已有页面风格一致
