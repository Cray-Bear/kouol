# I-008 · P0 主页开发 — 治理流程演练

**状态：** 🔀 已合并 → R-004
**优先级：** P0（最高）
**来源：** 用户决策 — "第一阶段只想做主页，专注一个页面搞清楚流程"

## 想法描述

用主页 p001（Home Screen）走一遍完整的治理流程：I-xxx → R-xxx → 三方讨论 → technical.md → 任务拆解 → 开发 → 测试 → 审查 → 发布 → 审计。

不追求交付多少功能，重点是**让治理模型在实践中接受检验**。

## 为什么现在值得做

- **治理模型需要实战验证** — 七权协作刚建立，纸上谈兵不如一个真实需求
- **主页是入口** — 首页是用户看到的第一个页面，做好了后面有基础
- **风险最小** — 一个页面，复杂度可控，适合当"演习场"
- **教学价值** — 完整走一遍流程，暴露问题，迭代治理模型

## 范围

- p001 首页（Figma 已有设计）
- 基础设施：路由系统、底部导航组件（为后续页面打基础）
- 走完治理全流程，产出审计报告

## 不在范围

- 其他页面（p002-p009）→ 等 P0 流程跑通后再安排
- 组件库抽象 → P2
- 真实后端数据 → 用 mock 数据即可

## 验收标准

- [ ] 主页在模拟器/设备上可运行
- [ ] 视觉还原 Figma 设计稿
- [ ] 治理全流程走通（有 discussion.md、log.md、test.md、review.md、审计报告）
- [ ] 治理模型经过实践检验，产出改进建议
