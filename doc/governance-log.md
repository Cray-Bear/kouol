## 2026-07-19 — 七权协作模型建立

### 背景
- 项目从"单 Agent 做所有事"进化到"多 Agent 分工协作"
- 用户明确提出：三权分离 → 七权协作 → 奖惩机制 → 完全自动化

### 演进过程
1. **三权分离**：议（设计）→ 行（实施）→ 审（验证）
2. **增加战略者**：用户指出"没有想法需求池会枯竭"
3. **增加监督和奖惩**：用户指出"需要能监督到各个 agent"
4. **最终形态**：策 → 议 → 行 → 审 → 督 + 用户裁决 + 记分板反馈

### 费曼输出标准（第三个思想体系）

来源：[feynman-skill](https://github.com/alchaincyf/feynman-skill)

**核心公式：** 具体例子开场 + 不用术语 + 主动标记不确定性 + 演示替代论证

融入方式：不独立成章节，直接嵌入每个角色的输出格式中作为质量注释。

| 角色 | 融入位置 | 关键原则 |
|------|---------|---------|
| Strategist | 想法格式 | 不用术语、从具体场景开始、诚实标记不确定性 |
| Architect | 技术方案格式 | 命名≠理解、先给场景再抽象 |
| PM | 需求描述格式 | 货物崇拜检测、用户能看懂吗？ |
| Developer | 日志记录格式 | 命名表达意图、演示>论证、标记不确定性 |
| Designer | 日志条目格式 | 具象化思考、不用设计术语解释决策 |
| Tester | Bug 列表格式 | O 型环范式：10 秒可复现演示 |
| Reviewer | 改进建议格式 | 反问句替代感叹句、反自欺 |
| Auditor | 问题发现格式 | 货物崇拜检测、直接验证、反身份固化 |
